# Reserved-Name Policy

## Scope

The names **AEP**, **Agent Element Protocol**, **AEP-compliant** and **dynAEP** are reserved names managed by New Lisbon Agency (NLA). This policy describes permitted and prohibited uses of these names. It is separate from, and additional to, the Apache License 2.0 that governs the source code in this repository.

Apache 2.0 grants you rights to use, modify and redistribute the code. It explicitly does **not** grant rights to use the project names on your fork or derived work. This file documents how those names may be used.

## Permitted Uses

You may use the reserved names when:

- Referring to this project or its documentation in articles, papers, tutorials, presentations or social posts.
- Describing your work as "compatible with AEP" or "compatible with the AEP specification" if it actually is.
- Claiming "AEP-compliant" status **only after** your implementation has passed the published conformance test suite and submitted a conformance report to the public registry.
- Linking to this repository or the official documentation.

## Prohibited Uses

You may not use the reserved names when:

- Forking this code and shipping your fork under the same name. Rename your fork.
- Claiming "AEP-compliant" status without a passing conformance report on file.
- Naming a competing product or service in a way likely to cause confusion with this project.
- Registering domain names, package names or social accounts that suggest official affiliation.

## Enforcement

New Lisbon Agency has filed protection applications for these names in multiple jurisdictions. Misuse of the reserved names may be addressed through standard legal processes. Good-faith questions about permitted use can be directed to the repository maintainer via issue.

## Conformance Registry

A public conformance registry will be published at `github.com/thePM001/aep-conformance` once the test suite ships. Until then, "AEP-compliant" claims are accepted on good faith for early adopters who document their compliance approach in a public README.

## Changes to This Policy

This policy may be updated. Existing uses consistent with a prior version of the policy remain permitted for 90 days after any change.
